# Design-Pattern

커널아카데미 부트캠프 디자인 패턴 스터디 레포지토리

## 정책

```text
1. 커밋 관련
    - 커밋시 메시지는 "{패턴명}: {작업내용}" 으로 작성
    - 커밋하기 전 프리커밋 검사 필수
2. 브랜치 관련
    - 브랜치명은 "{패턴명}/{작업자이름} 으로 작성
3. PR 관련
    - PR 명은 "PR {패턴명} {작업자이름}"
    - PR시 리뷰어는 모든 스터디원을 태그
4. 코드 리뷰 관련
    - 토끼의 리뷰와 리뷰어들의 리뷰를 모두 참고하여 코드 수정
    - 리뷰어들은 읽어보고 댓글이나 어프로브 전달
5. 머지 관련
    - 머지시 기본값으로 머지하고, 각자가 자신의 것을 머지함
```

## 커밋 전 필수 검사

```bash
pre-commit run --all-files
```
